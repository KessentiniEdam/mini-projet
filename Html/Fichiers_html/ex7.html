Second document HTML

Quelques remarques

  1. Sauf cas exceptionnels, la mise en forme dans le document HTML source n'est pas prise en compte, en particulier les retours à la ligne.
  2. Le cas de la balise <img> est particulier, celle-ci ne comporte pas de contenu et n'a donc pas de balise fermante.
  3. Certains caractères spéciaux sont codés par une entité dans le code source. Ces entités commencent pas le symbole "&" et se terminent par ";". Les entités peuvent se référencer par un nom ou par le code hexadécimal du caractère en UTF-8, ce code est alors précédé de #x.

    En XHTML5, seules quelques entités nommées sont reconnues, notamment &lt; et &gt; pour les caractères "<" et ">". Pour les autres il faut utiliser le code UTF-8.

    Voici quelques autres exemples d'entités avec leur nom HTML et leur code UTF-8 :

    • &nbsp; ou &#xA0; pour un caractère blanc insécable " ", nécessaire pour respecter les règles typographiques, par exemple avant un ":"
    • &laquo; ou &#xAB; et &#xBB;/&#187; pour les caractères "«" et "»"
    • &euro; ou &#x20AC; pour le caractère "€"
    • &frac12; ou &#xBD; pour le caractère "½"
    • etc.

    Il en existe beaucoup d'autres, vous pouvez en trouver une liste complète sur la page de référence du W3C (ou encore ce document en travail), ou encore ce document pour une présentation par catégorie.

  4. Les commentaires s'écrivent dans le code source entre les balises spéciales <!-- et -->. Cette partie du document est ignorée par le navigateur.

Les balises

Les balises HTML sont nombreuses. En plus de permettre la structuration du texte, les balises ont une certaine sémantique. Celle-ci permet de caractériser le contenu de la balise. Le choix de l'utilisation d'une balise plutôt qu'une autre se fait donc en fonction du statut son contenu. On trouve ainsi des balises pour représenter des titres, des paragraphes, des portions de texte à mettre en évidence, d'autres correspondant à des dates ou des abréviations, etc.

Même si à toute balise correspond un style visuel par défaut utilisé par le navigateur, le choix d'une balise ne doit pas être fait en fonction de ce style visuel. Nous verrons lorsque nous étudierons les feuilles CSS que cet aspect visuel peut être modifié à tout moment.

Le choix de la balise dépend donc bien de sa sémantique et non de son rendu graphique.

Les éléments que l'on peut trouver dans l'élément body sont des éléments de contenu de flux. On y trouve (cf. figure ci-dessous) :

  • des éléments de contenu de section,
  • des éléments de contenu de titre dont les balises se nomment <h1> à <h6>,
  • des éléments de contenu de phrasé, dont le texte simple , qui mis en séquence constituent des paragraphes. Par exemple <strong>, <a>, <abbr>, etc. On parle également d'éléments ligne pour de tels éléments.
  • des éléments structurants destinés à accueillir du contenu phrasé (<p>, <pre> par exemple), ou d'autres éléments avec certaines restrictions, comme l'élément <ul> avec les éléments <li>. On trouve parfois le terme d'éléments bloc pour parler de tels éléments.
  • des éléments destinés à accueillir des contenus particuliers comme par exemple les éléments <img> ou <canvas>.
